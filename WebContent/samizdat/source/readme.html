JConfig Source Code Readme

This file has basic information on this distribution.

Getting Started

In order to best understand how to use this distribution, it is suggested that you read the documentation in the following order:

  1. Read the manifests to familiarize yourself with the files included in this distribution:
    • Overview of all files.
    • List of the native code files common to all platforms.
    • List of the files used by the Windows native code.
    • List of the files used by the Mac native code.
    • List of the files used by the Linux native code.

  2. Read the installation notes appropriate for your platform:
    • Windows installation.
    • Mac installation.
    • Linux installation.
    • OSX installation.
    It is suggested that you make Windows your 'home base', copying files to Mac and Linux only to compile the native code.

  3. Read the additional native code information, in order to get a general idea of how various macros and defines are used to put together the different builds. You can refer to these files later as you write your own code.
    • Information on the macros used to minimize JNI/RNI/JRI differences.
    • Information on the string classes used in native code.
    • Information on miscellaneous macros and defines used in native code.

  4. In order to get an idea of how calls to the JConfig public API are handled, read the two walkthroughs.
    • A walkthrough of a call to FileRegistry.initialize().
    • A walkthrough of a call to FileRegistry.getMainMonitor().

  5. Review the Java and native code class documentation in order to determine how best to modify JConfig to suit your purposes.
    • The Java class documentation.
    • C++ classes common to all platforms.
    • C++ classes used on Windows.
    • C++ classes used on Mac.
    • C++ classes used on Linux.

The navigation bar at the top of this page has links to the documentation included with this distribution. This navigation bar appears on most of the other documentation as well.

For legal, technical, and contact information, see the main readme.html file.
