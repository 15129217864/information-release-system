Mac Installation

This file contains installation and setup notes for Mac.

Installation

You can leave the source folder inside the samizdat folder, or you can place it in another location, such as inside your CodeWarrior folder if it's more convenient.

The source folder contains two Stuffit archives which contain the Mac-specific files: 

    mac_Project.sit               a folder containing the CodeWarrior project
    mac_JConfig_rsrc.sit          the JConfig resource file (JConfig.rsrc)

To install, unstuff each of these files in place: i.e., unstuff both of them into the 'source' folder.

NOTE: If Stuffit does not recognize the .sit files as Stuffit archives, you'll need to change the creator of each of these files to 'SIT!', and the type of each of these files to 'SITD'.

NOTE: We recommend that you use Windows as your 'homebase', and only transfer files to the Mac in order to compile the Mac shared libraries. So, the CodeWarrior project file included with this distribution only compiles the native code, not Java.

Requirements

To compile the Mac shared libraries, you need:

  • Metrowerks CodeWarrior, Pro2 or later
    MPW might also work, however, this has not been tested.
    Make sure that you installed Java support when you installed CodeWarrior; the CW builds of the shared libraries need the JNI header files included with the CW Java installation.

  • the Internet Config SDK
    The JConfig binaries were compiled using ICProgKit1.3.
    You can get this from www.tolstoy.com/samizdat/jcextras.html
    The CodeWarrior project assumes that this is stored in the source folder (i.e., source:ICProgKit1.3:APIs, etc.)
    This SDK is used by both the CW and MRJ builds.

  • MoreFiles
    The JConfig binaries were compiled using MoreFiles 1.4.5.
    You can get this from www.tolstoy.com/samizdat/jcextras.html
    The CodeWarrior project assumes that this is stored in the source folder (i.e., source:MoreFiles 1.4.5)
    This SDK is used by both the CW and MRJ builds.

  • the FindIcon library
    The JConfig binaries were compiled using this library from 10/28/98.
    You can get this from www.tolstoy.com/samizdat/jcextras.html
    The CodeWarrior project assumes that this is stored in the source folder (i.e., source:Find_icon folder)
    This SDK is used by both the CW and MRJ builds.

  • the MRJ SDK version 1.0.2 or 1.5
    This is required for the JRI header files.
    The CodeWarrior project assumes that this is stored in Macintosh HD:MRJ 1.0.2 SDK
    If you're using 1.5, or if you have this SDK at another location, you'll need to change this path in the CW Project Settings/Access Paths dialog.
    You can get this from http://developer.apple.com/java/text/download.html
    This SDK is only used by the MRJ build.

To compile the Java code, you'll need a Java compiler, such as Metrowerks or the tools included with MRJ. Since we only support compiling the Java code on Windows, you'll need to create a project yourself.

Compiling

You'll need to edit the preferences of the CodeWarrior project to reflect the locations where you've stored the Internet Config and MRJ SDKs.

To compile, open the CodeWarrior project, and select the 'BuildAll' target. This will create four shared libraries:

  • JConfigMRJ.shlb: for use with MRJ
  • JConfigCW.shlb: for use with the CW VM (standard)
  • JConfigCWJIT.shlb: for use with the CW VM (JIT)
  • JConfig.shlb: merges the three shared libraries listed above

JConfig.shlb is the standard shared library distributed with JConfig, however, if you're only running on one VM, you might wish to use one of the other libraries.

Notes

IMPORTANT NOTE: Don't modify the relative locations of files or folders within the distribution; doing so may require editing the project file.
